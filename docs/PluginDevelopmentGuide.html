PowerDesk Plugin Development Guide

Version 2.0 — Build extensions for the file explorer you wish Windows came with.

1. Quick Start

A PowerDesk plugin is a folder containing a manifest.json and an index.js entry file. Plugins live in:

%APPDATA%\PowerDesk\plugins\your-plugin-id\

Each plugin folder looks like this:

my-plugin/
  manifest.json
  index.js

manifest.json

{
  "id": "my-plugin",
  "name": "My Awesome Plugin",
  "version": "1.0.0",
  "description": "A brief description of what this plugin does.",
  "author": "Your Name"
}
FieldRequiredDescription
idYesUnique identifier. Use lowercase + hyphens.
nameYesDisplay name shown in Settings.
versionYesSemantic version string (e.g. "1.0.0").
descriptionYesShort description shown in the plugin list.
authorYesYour name or organization.
permissionsNoArray of permission strings (reserved for future use).

index.js

Your entry file must export an activate function (or default export). This function receives the Plugin API object:

module.exports = { activate(api) {
  // Register your extension points here
  api.log.info('Plugin loaded!')
}}
Note: Plugins run in the renderer process. Use CommonJS (module.exports) syntax. React, ReactDOM, and Lucide icons are available as globals via require() shims.

2. The Plugin API

The api object passed to activate() provides access to all extension points. The API is scoped to your plugin ID.

2.1 api.store — Zustand Store

MethodDescription
getState()Read the full app state (one-shot snapshot).
setState(partial)Merge partial state into the store.
subscribe(listener)Subscribe to store changes. Returns an unsubscribe function. The listener receives (state, prevState). Use this for reactivity — getState() is a snapshot.
Reactivity: getState() returns a snapshot at one point in time. To react to changes (e.g. user navigates to a different folder), use subscribe(): 
const unsub = api.store.subscribe((state, prev) => {
  if (state.activeTabId !== prev.activeTabId) {
    const tab = state.tabs.find(t => t.id === state.activeTabId)
    api.log.info(`Navigated to: ${tab?.path}`)
  }
})
// Later: unsub()

2.2 api.ui — UI Extensions (16 Methods)

MethodDescription
registerSidebarPanel(panel)Add a collapsible panel to the sidebar below built-in sections.
registerToolbarButton(btn)Add a button to the TopBar toolbar row.
registerStatusBarEntry(entry)Add an entry to the status bar at the bottom.
registerContextMenuItem(item)Add an item to the file right-click context menu.
registerBackgroundContextMenuItem(item)Add an item to the background (folder area) right-click menu.
registerSettingsTab(tab)Add a new tab to the Settings panel.
registerPreviewHandler(handler)Register a custom preview for specific file extensions.
registerView(view)Register a custom view that can be opened as a main area panel.
registerTabType(tabType)Register a custom tab type with its own icon and component.
registerTreeDataProvider(provider)Register a tree data provider that renders a tree view in the sidebar.
registerFileDecorationProvider(provider)Register a file decoration provider (color, badge, tooltip on files).
registerTheme(theme)Register a custom theme with color overrides.
registerTopBarItem(item)Register a custom item in the top bar (next to tabs).

2.3 api.events — Event System

Subscribe to app-wide events. All listeners receive the event payload. Returns an unsubscribe function.

EventPayloadDescription
didNavigate{ path, tabId, previousPath }Fired when a tab navigates to a new folder.
didTabChange{ tabId, previousTabId }Fired when the active tab changes.
didFileOperation{ type, path, destination? }Fired after a file operation (copy, move, delete, rename, create).
didSettingsChange{ key, value }Fired when any setting changes.
appReadyvoidFired once after all plugins are loaded and the app is ready.
api.events.on('didNavigate', (payload) => {
  api.log.info(`Navigated to: ${payload.path}`)
})

api.events.on('didFileOperation', (payload) => {
  if (payload.type === 'delete') api.log.info(`Deleted: ${payload.path}`)
})

2.4 api.hooks — Before/After Hooks

Intercept or modify file operations. before hooks can cancel operations by throwing or returning false.

MethodDescription
before(operationId, callback)Run before a file operation. Throw or return false to cancel.
after(operationId, callback)Run after a file operation completes.

Available hook IDs: deleteFile, renameFile, createFolder, createFile, copyFile, moveFile.

api.hooks.before('deleteFile', (params) => {
  // params: { path, permanent }
  if (params.path.includes('important')) {
    api.log.warn('Blocked deletion of important file!')
    return false // Cancel the operation
  }
})

api.hooks.after('renameFile', (params) => {
  api.log.info(`Renamed ${params.oldPath} → ${params.newPath}`)
})

2.5 api.commands — Command Palette

MethodDescription
register(cmd)Add a command to the Command Palette (Ctrl+Shift+P).
api.commands.register({
  id: 'open-terminal-here',
  label: 'Open Terminal Here',
  category: 'Tools',
  shortcut: 'Ctrl+Shift+`',
  action: () => {
    const state = api.store.getState()
    const tab = state.tabs.find(t => t.id === state.activeTabId)
    if (tab?.path) api.ipc.invoke('terminal-exec', tab.path, 'cd')
  },
})

2.6 api.shortcuts — Keyboard Shortcuts

MethodDescription
register(binding)Register a global keyboard shortcut.
api.shortcuts.register({
  id: 'focus-search',
  keys: 'Ctrl+Shift+F',

2.7 api.fs — File System

MethodDescription
readDir(dirPath)List files in a directory. Returns FileItem[].
readFile(filePath)Read a file as text. Returns { content: string, size: number }.
writeFile(filePath, content)Write text to a file.
stat(filePath)Get file stats (size, dates, isDirectory).

2.8 api.ipc — Electron IPC

MethodDescription
invoke(channel, ...args)Call an IPC handler in the main process.
on(channel, callback)Listen for IPC messages from main process.
send(channel, ...args)Send a message to the main process.

2.9 api.log — Logging

MethodDescription
info(message)Log an informational message. Prefixes with [Plugin:your-id].
warn(message)Log a warning.
error(message)Log an error.

3. Extension Points — All 16 Types

3.1 Sidebar Panel

Add a collapsible panel below the built-in sidebar sections (Tags, Drives, etc.):

const { createElement: h } = require('react')

module.exports = { activate(api) {
  function MyPanel() {
    const files = api.store.getState().files
    const dirs = files.filter(f => f.isDirectory).length
    return h('div', { style: { padding: '8px 12px', fontSize: 12 } },
      h('div', { style: { color: 'var(--accent)', fontWeight: 600, marginBottom: 4 } }, 'Folder Stats'),
      h('div', { style: { color: 'var(--text-secondary)' } },
        `${dirs} directories, ${files.length - dirs} files`
      )
    )
  }

  api.ui.registerSidebarPanel({
    id: 'folder-stats',
    label: 'Folder Stats',
    icon: h('span', null, '\u{1F4CA}'),
    component: MyPanel,
    position: 100,
  })
}}

3.4 File Context Menu Item

Add an item to the right-click menu when selecting files/folders:

module.exports = { activate(api) {
  api.ui.registerContextMenuItem({
    id: 'copy-path',
    label: 'Copy Full Path',
    icon: '📋',
    onClick: () => {
      const state = api.store.getState()
      if (state.contextTarget?.item) {
        navigator.clipboard.writeText(state.contextTarget.item.path)
        api.log.info('Path copied!')
      }
    },
    condition: (state) => !!state.contextTarget?.item,
    dividerAfter: true,
    position: 5,
  })
}}

3.10 Tree Data Provider

Register a tree view that appears in the sidebar. The provider supplies nodes, and the sidebar renders them as a collapsible tree:

const { createElement: h } = require('react')

module.exports = { activate(api) {
  api.ui.registerTreeDataProvider({
    id: 'project-structure',
    label: 'Project',
    getTreeNodes: () => {
      return [
        { id: 'src', label: 'src', icon: '📁', children: [
          { id: 'components', label: 'components', icon: '📁', children: [
            { id: 'app', label: 'App.tsx', icon: '📄', filePath: '/src/components/App.tsx' },
          ]},
          { id: 'index', label: 'index.ts', icon: '📄', filePath: '/src/index.ts' },
        ]},
        { id: 'package', label: 'package.json', icon: '📄', filePath: '/package.json' },
      ]
    },
    onNodeClick: (node) => {
      if (node.filePath) api.log.info(`Clicked: ${node.filePath}`)
    },
    position: 10,
  })
}}

Tree nodes can have children for nested trees. Click events pass the full node object.

3.11 File Decoration Provider

Add visual decorations (color, badge, tooltip) to files in the grid and list views:

module.exports = { activate(api) {
  api.ui.registerFileDecorationProvider({
    id: 'git-status',
    getFileDecoration: (file) => {
      if (file.name.endsWith('.ts') || file.name.endsWith('.tsx')) {
        return {
          color: '#3178c6',     // TypeScript blue
          badge: 'TS',          // Small badge text
          tooltip: 'TypeScript file',

        return {
          color: '#4caf50',
          badge: 'TEST',
          fontWeight: 'bold',
          strikethrough: false,
        }
      }
      return undefined // No decoration
    },
  })
}}

4. Accessing App State

The Zustand store contains the full application state. Here are the key slices:

const state = api.store.getState()

// Current tabs
state.tabs            // Tab[]
state.activeTabId     // string
// NOTE: state.activeTab does NOT exist!
// Derive it: tabs.find(t => t.id === activeTabId)

// Files in current view
state.files           // FileItem[]
state.focusedFileIndex // number

// Sidebar
state.sidebarOpen     // boolean
state.initialDirs     // Record<string, string>

// UI settings
state.ui              // UiSettings (accentColor, fontSize, theme, etc.)
state.settings        // AppSettings (defaultView, confirmDelete, etc.)

// Clipboard
state.clipboardItems  // ClipboardItem[]

// Transfers
state.transfers       // Transfer[]

// Tags
state.allTags         // string[]
state.fileTags        // Record<string, string[]>

// Favorites
state.favorites       // FavoriteItem[]
Important: There is no state.activeTab property. The store has tabs[] and activeTabId separately. Always derive the active tab:

const tab = state.tabs.find(t => t.id === state.activeTabId)
Navigation: To navigate to a folder, call api.store.getState().navigateTo('/path/to/folder').

5. Best Practices

  • Use unique IDs: Prefix your plugin ID with your org name (e.g. acme-file-tools) to avoid collisions.
  • Use position: Lower values appear first. Default is 50. Use 10-20 for high priority, 80-100 for low priority.
  • Subscribe for reactivity: Use api.store.subscribe() in components that need to re-render on state changes. getState() is a one-shot snapshot.
  • Graceful degradation: Check for API availability before calling IPC methods. Not all channels may be present.
  • Clean up: Store unsubscribe functions from subscribe() and events.on() for cleanup.
  • CSS Variables: Use the app's CSS variables (var(--accent), var(--text-secondary), etc.) so your UI matches the active theme.
  • React Components: Use createElement (aliased as h) or JSX. React 18 is available globally.
  • Don't block: Never run synchronous heavy computation in the renderer. Use async operations and setTimeout.
  • Hook safety: Before hooks can cancel operations. Only block operations you need to — don't block everything.

6. Plugin Lifecycle

  1. Discovery: On app launch, PowerDesk scans %APPDATA%\PowerDesk\plugins\ for folders containing manifest.json.
  2. Loading: Each valid plugin's index.js is loaded via new Function() with a CJS shim providing require('react') and require('lucide-react').
  3. Activation: The activate(api) function is called. Register your extension points here.
  4. Runtime: Your registered components and handlers remain active until the app closes or the plugin is disabled. Events fire for navigation, file operations, and tab changes.
  5. Disable/Enable: Users can toggle plugins on/off in Settings > Plugins. Disabled plugins have their extensions hidden but remain loaded in memory.

7. Full Example

A plugin that adds a sidebar panel, file decorations, a background context menu item, listens to events, and registers a custom view:

// manifest.json
{
  "id": "file-insights",
  "name": "File Insights",
  "version": "1.0.0",
  "description": "Shows file analytics, decorations, and a dashboard view.",
  "author": "PowerDesk Community"
}

// index.js
const { createElement: h } = require('react')

module.exports = { activate(api) {

  // ─── Sidebar Panel ────────────────────────────────────────────
  function InsightsPanel() {
    const [data, setData] = h('div', { style: { padding: 12 } }, 'Loading...')
    h('useEffect', () => {
      const unsub = api.store.subscribe((state) => {
        const files = state.files
        const totalSize = files.reduce((sum, f) => sum + (f.size || 0), 0)
        setData(
          h('div', { style: { fontSize: 12 } },
            h('div', { style: { fontWeight: 600, marginBottom: 8, color: 'var(--accent)' } }, 'Insights'),
            h('div', null, `Files: ${files.length}`),
            h('div', null, `Total size: ${(totalSize / 1024).toFixed(1)} KB`)
          )
        )
      })
      return unsub
    }, [])
    return data
  }

  api.ui.registerSidebarPanel({
    id: 'insights',
    label: 'Insights',
    icon: h('span', null, '📊'),
    component: InsightsPanel,
    position: 100,
  })

  // ─── File Decorations ─────────────────────────────────────────
  api.ui.registerFileDecorationProvider({
    id: 'insights-deco',
    getFileDecoration: (file) => {
      if (file.size > 1024 * 1024) {
        return { color: '#ff6b6b', badge: 'L', tooltip: 'Large file' }
      }
      return undefined
    },
  })

  // ─── Background Context Menu ──────────────────────────────────
  api.ui.registerBackgroundContextMenuItem({
    id: 'open-insights',
    label: 'View Insights',
    icon: '📊',
    onClick: () => api.log.info('Opening insights dashboard...'),
    position: 100,
  })

  // ─── Event Listeners ──────────────────────────────────────────
  api.events.on('didNavigate', (payload) => {
    api.log.info(`Folder changed: ${payload.path}`)
  })

  api.events.on('didFileOperation', (payload) => {
    api.log.info(`File op: ${payload.type} — ${payload.path}`)
  })

  // ─── Custom View ──────────────────────────────────────────────
  function DashboardView() {
    return h('div', { style: { padding: 24 } },
      h('h2', { style: { color: 'var(--accent)' } }, 'Insights Dashboard'),
      h('p', { style: { color: 'var(--text-secondary)', marginTop: 8 } },
        'Custom view rendered in the main area.')
    )
  }

  api.ui.registerView({
    id: 'insights-dashboard',
    title: 'Insights',
    icon: '📊',
    component: DashboardView,
  })

  // ─── Command ──────────────────────────────────────────────────
  api.commands.register({
    id: 'open-insights-dashboard',
    label: 'Open Insights Dashboard',
    category: 'Tools',
    action: () => api.store.getState().openPluginView('insights-dashboard'),
  })

  api.log.info('File Insights plugin activated!')
}}

8. Troubleshooting

  • Plugin not showing up? Check the browser console (F12) for errors. Ensure your manifest.json has all required fields. Check the Plugins tab in Settings.
  • Component not rendering? Make sure your activate() function is being called. Check api.log.info('loaded!').
  • Store not updating reactively? Don't rely on getState() for ongoing updates. Use subscribe() to get notified of changes.
  • No state.activeTab? The store has tabs[] and activeTabId. Derive it: tabs.find(t => t.id === activeTabId).
  • Keyboard shortcut not working? Make sure the key string matches: Ctrl+Shift+K, Alt+F5, etc. Check for conflicts with built-in shortcuts.
  • Context menu not appearing? Check the condition function if you set one — it must return true.
  • Sidebar too wide / layout broken? Plugin sidebar panels are constrained by the sidebar width. Use overflow: auto on scrollable content.
  • Hook not firing? Ensure the hook ID matches exactly: deleteFile, renameFile, createFolder, createFile.
